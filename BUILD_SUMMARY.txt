PROJECT OVERVIEW
================================================================================

A production-ready Next.js App Router application that dynamically generates
personalized Digital Sales Rooms powered by Supabase.

This replaces the static Novi DSR (in novi-dsr/) with a flexible, database-driven
system where each prospect slug maps to a custom DSR pulled from Supabase.

================================================================================
DATABASE REQUIREMENTS
================================================================================

The app expects a Supabase table named "digital_sales_rooms" with columns:

Required (minimum):
  - id (UUID, primary key)
  - slug (TEXT, unique)
  - prospect_name
  - prospect_contact
  - prospect_title

Optional but supported:
  - meeting_date, meeting_title, meeting_duration
  - attendees (JSONB array)
  - coram_team (JSONB array)
  - environment (JSONB object)
  - outcomes (JSONB array)
  - call_highlights (JSONB array)
  - milestones (JSONB array)
  - resources (JSONB array)
  - references_data (JSONB array)
  - grants (JSONB array)
  - pricing (JSONB object with access_code)
  - status (active/draft/archived)
  - created_at, updated_at (timestamps)

See SUPABASE_MIGRATION.md for complete schema and sample data.
